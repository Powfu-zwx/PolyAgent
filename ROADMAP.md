# Roadmap

## Now

- Publish the first public release package with stable README, metadata, screenshots, and release notes.
- Keep the core four capabilities visible and easy to demo: QA, summary, writing, and guidance.
- Improve first-visit conversion on GitHub with clearer positioning and reusable launch assets.

## Next 2-4 Weeks

- Record a 30-60 second walkthrough video or GIF for the repository and social channels.
- Add FAQ and troubleshooting docs based on the first round of user feedback.
- Enable GitHub Discussions and collect scenario requests, issue reports, and showcase examples.
- Publish at least one Chinese article and one English article that explain the project and its architecture.

## Product Iteration

- Make prompts and example data easier to switch across industries and languages.
- Expand configuration options for model providers, routing behavior, and deployment setup.
- Improve packaged installation and self-hosting guidance for new adopters.

## Community and Growth

- Share focused content around each core capability instead of only the full stack.
- Turn real use cases into reusable prompt packs and demo scenarios.
- Maintain a visible update rhythm through release notes, roadmap updates, and issue triage.
